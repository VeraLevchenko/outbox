---

### 5. РОЛИ И ПРАВА

#### Роли
- `director` — руководитель
- `head` — начальник отдела

#### Колонки Kaiten

**Director**
- **источник:** «На подпись»
- **действия:**
  - выбрать исполнителя (обязательно)
  - зарегистрировать + подписать
  - вернуть на доработку
  - пропустить
- **Маршруты:**
  - подписать → «Отправка»
  - вернуть → «Проект готов. Согласование начальника отдела»

**Head**
- **источник:** «Проект готов. Согласование начальника отдела»
- **действия:**
  - подписать → «На подпись»
  - вернуть → «В работе»

### 6. ЛОГИКА ОЧЕРЕДИ

- **Получение карточек:** polling Kaiten каждые 5–10 секунд
- Показывать по одной карточке
- После действия → следующая
- После перезагрузки → первая
- Блокировки карточек НЕ нужны

---

### 8. ПРОЦЕСС «ЗАРЕГИСТРИРОВАТЬ И ПОДПИСАТЬ» (СТРОГО)

**Последовательность НЕ МЕНЯТЬ:**

1. Пользователь выбирает исполнителя
2. Получить следующий `outgoing_no` из PostgreSQL (max + 1)
3. `outgoing_date` = текущая дата
4. В DOCX заменить плейсхолдеры:
   - `{{outgoing_no}}`
   - `{{outgoing_date}}`
   - `{{stamp}}` — во всех вхождениях
5. DOCX → PDF
6. Подписать PDF через CryptoPro
7. Создать папку:
   ```
   /mnt/doc/Исходящие/<outgoing_no>/
   ```
8. Сохранить:
   - подписанный PDF
   - файл подписи
   - все приложения
9. Записать данные в БД
10. Переместить карточку Kaiten (с комментарием или без)

### Для монтирования

# 0. Важно для WSL: после перезагрузки компьютера, если шара не подхватится автоматически, выполни:
sudo mount -a

# 1. Создать точку монтирования (если нет папки)
sudo mkdir -p /mnt/doc

# 2. Смонтировать шару вручную
sudo mount -t cifs //10.1.10.30/doc /mnt/doc -o username=lvs,password=123,iocharset=utf8,vers=3.0

# 3. Проверить, что работает
ls -la /mnt/doc/Входящие/

# 4. Добавить в fstab для автомонтирования
echo '//10.1.10.30/doc /mnt/doc cifs username=lvs,password=123,iocharset=utf8,vers=3.0,nofail,_netdev 0 0' | sudo tee -a /etc/fstab

# 5. Если после перезагрузки отвалилось
sudo mount -a
